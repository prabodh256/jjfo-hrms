WORKFLOWS The signature flows

Onboarding — submission & document approval

EDITABLE
Draft / Returned
Employee fills education, experience & uploads docs
→
LOCKED
Submitted
Needs all mandatory docs · cells lock on submit
→
LOCKED
Approved
Admin reviews attachments & approves — no more edits
Employee can edit Locked Admin can return a submitted/approved record → back to editable with a note. Mandatory docs configurable by admin.

Leave approval — up the reporting chain

REQUEST
Employee applies
Duration decides the depth
→
LEVEL 1
Direct manager
Approves with the "approve leaves" capability
→
LEVEL 2 · >5 DAYS
Manager's manager
Second sign-off only when > 5 days
→
DONE
Approved
Balance updates
≤ 5 days → 1 level · > 5 days → 2 levels (clamped to how many managers exist). Each step needs the right approver; admin can finalize any step.

Delegated permissions — the subset rule

GRANTER
Person 1
Holds a set of modules + capabilities
→
CREATES
Person 1A
Gets a subset — e.g. "approve leaves" but not "create users"
→
BLOCKED
1A → 1A(a)
Without "create users", 1A cannot create further. Over-granting → 403
New users are placed under their creator in the hierarchy. A grant can never exceed the granter's own permissions.

FOUNDATION Security & architecture

Hardened authentication

bcrypt password hashing (cost 12)
JWT in httpOnly + Secure + SameSite cookies
Env-only JWT secret, fail-fast at boot
Login rate-limit (failed attempts only) + global limiter
zod input validation
Public registration removed
CSRF header required on all writes
Helmet headers · shared Prisma client
Password hashes never sent to the client
Server-enforced permission & lock checks

Architecture

React 19 + Vite SPA (react-router + zustand)
Express 5 + Prisma + SQLite
SQLite is the single source of truth
Vite proxies /api and /auth to the backend
Legacy localStorage monolith retired
Rich seed data across every module
Optimized: no N+1 on leaves, parallel dashboard counts
Atomic cascade delete · payroll batch preload
